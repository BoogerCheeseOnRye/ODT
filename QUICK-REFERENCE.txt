# TEoAAAG - Quick Reference Card

## One-Time Setup (5 minutes)

```powershell
# 1. Set Ollama environment (Windows)
Settings → Environment Variables
  ADD: OLLAMA_MODELS = E:\models
  ADD: OLLAMA_CACHE = E:\cache
  Restart Ollama

# 2. Create model directory
mkdir E:\models

# 3. Start Ollama
ollama serve

# 4. In NEW terminal, start dashboard
cd E:\dashboard-appv2
start-dashboard-isolated.bat
```

## Daily Startup

```batch
# Terminal 1: Ollama
ollama serve

# Terminal 2: Dashboard
cd E:\dashboard-appv2
start-dashboard-isolated.bat
```

Then open: **http://localhost:8080**

## Model Management

**Add Model:**
1. Copy `.gguf` file to `E:\models\`
2. Dashboard → "📦 Models" → "🔍 Scan"
3. Click "Import"

**List Models:**
1. Dashboard → "📦 Models"
2. Scroll to "Loaded Models"

**Delete Model:**
1. Delete `.gguf` from `E:\models\`
2. Restart Ollama

## Directories

| Path | Purpose | Contents |
|------|---------|----------|
| `E:\models` | Model storage | `.gguf` files |
| `E:\cache` | Ollama cache | Auto-managed |
| `E:\dashboard-appv2` | App folder | Code only |

## URL

- **Dashboard:** http://localhost:8080
- **Proxy:** http://localhost:9001
- **Ollama:** http://localhost:11434

## Security (What's Protected)

✓ App folder cannot store models
✓ Models must be in `E:\models`
✓ Path validation prevents escapes
✓ HTTP 403 if wrong path

## Common Tasks

| Task | Steps |
|------|-------|
| Change model | Chat selector in dashboard |
| View settings | ⚙️ Settings button |
| Scan new models | 📦 Models → 🔍 Scan |
| Clear chat | Chat panel → Clear button |
| Fullscreen preview | F key or corner button |
| Show stats | ⚙️ Stats toggle |

## If Something's Wrong

| Problem | Solution |
|---------|----------|
| "Models not found" | Check `E:\models` exists, rescan |
| "Ollama unavailable" | Start: `ollama serve` |
| "Port 9001 in use" | Kill other app using that port |
| "Forbidden model" | Verify model in `E:\models`, not app folder |
| Models disappeared | Restart Ollama |

## Environment Variables (PowerShell)

```powershell
# Check current settings
$env:OLLAMA_MODELS
$env:OLLAMA_CACHE

# Set for session
$env:OLLAMA_MODELS = "E:\models"
$env:OLLAMA_CACHE = "E:\cache"

# Verify
Get-ChildItem env:OLLAMA_*
```

## Key Points

1. **Models go in E:\models** - NOT in app folder
2. **Set OLLAMA_MODELS before Ollama starts** - Required for external directory
3. **Use startup scripts** - Auto-configures environment
4. **App folder stays clean** - Models isolated externally
5. **Validation prevents mistakes** - HTTP 403 if wrong path
