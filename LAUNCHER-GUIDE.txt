================================================================================
                           WHAT EACH STEP DOES
================================================================================

STEP 1: CHECK DEPENDENCIES
  • Verifies index.html exists
  • Verifies proxy.js exists
  • Checks for Dashboard.exe (warns if missing)
  • Exits if critical files missing

STEP 2: CLEAR OLD PROCESSES
  • Kills any existing node.exe processes
  • Kills any existing Dashboard.exe instances
  • Ensures clean startup

STEP 3: SCAN MODELS DIRECTORY
  • Creates E:\models if missing
  • Counts .gguf files
  • Lists each model with size
  • Shows: Qwen_Qwen3.5-9B-Q4_K_M.gguf (8.45 GB)

STEP 4: VERIFY LIBRARY SYSTEM
  • Creates lib\ directory if missing
  • Checks for lib\bundle.lib.js
  • Initializes library system if needed

STEP 5: START PROXY SERVER
  • Starts node proxy.js in background (hidden window)
  • Waits for proxy to initialize
  • Verifies listening on port 9001
  • Confirms ready for API calls

STEP 6: VERIFY CONFIGURATION
  • Checks dashboard UI files
  • Checks proxy configuration
  • Verifies library bundle
  • Shows all configuration status

STEP 7: SYSTEM STATUS CHECK
  • Runs 3-point diagnostic:
    ✓ Dashboard File OK
    ✓ Proxy Running OK
    ✓ Models Directory OK
  • Reports pass/fail count

STEP 8: LAUNCH DASHBOARD
  • Launches Dashboard.exe with URL
  • Falls back to default browser if needed
  • Shows completion screen with stats

================================================================================
                          TROUBLESHOOTING
================================================================================

ISSUE: "PowerShell not found" error
SOLUTION:
  • Use: dashboard-init.bat instead
  • Or install PowerShell

ISSUE: Progress bar not showing
SOLUTION:
  • Check PowerShell version (need 5.0+)
  • Use: dashboard-init.bat as fallback

ISSUE: Dashboard.exe not launching
SOLUTION:
  • Verify Dashboard.exe in E:\dashboard-appv2\
  • Check execution permissions
  • Browser fallback will work instead

ISSUE: Proxy won't start
SOLUTION:
  • Check Node.js installed: node --version
  • Verify proxy.js exists
  • Check port 9001 not in use: netstat -ano | find "9001"

ISSUE: Models not found
SOLUTION:
  • Verify E:\models directory exists
  • Check .gguf files are in E:\models\
  • Try manual scan in dashboard (📦 Models button)

ISSUE: Setup window closes too fast
SOLUTION:
  • Edit: dashboard-init.ps1
  • Change: Start-Sleep -Seconds 2 (increase number)
  • Or: Add more Start-Sleep calls between steps
