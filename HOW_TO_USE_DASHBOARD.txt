TEoAAAG QUICK START GUIDE
=========================

WHAT YOU NEED
=============

1. Node.js installed
   - Download from nodejs.org
   - Add to PATH during install

2. Ollama running
   - Download from ollama.ai
   - Run: ollama serve
   - Make sure port 11434 is open

3. This dashboard
   - Extract TEoAAAG-Dashboard-Standalone.zip
   - Or use E:\dashboard-app\


LAUNCH THE DASHBOARD
====================

Option A: Quick Launch (Windows)
  1. Go to E:\ drive
  2. Double-click dashboard.bat
  3. Wait 2 seconds
  4. Dashboard opens in browser
  5. Done

Option B: From Dashboard App Folder
  1. Go to E:\dashboard-app\
  2. Double-click launch.bat
  3. Browser opens automatically
  4. Done

Option C: Manual Launch
  1. Open terminal/PowerShell
  2. cd E:\dashboard-app
  3. node proxy.js
  4. Open browser: file:///E:/dashboard-app/index.html
  5. Start typing in chat


USING THE DASHBOARD
===================

Layout (3 columns):

LEFT PANEL
  - Chat messages from you
  - Chat responses from Qwen AI
  - Type here, hit Ctrl+Enter or Shift+Enter to send

CENTER PANEL
  - Status and output messages
  - Shows what's running
  - Displays errors if any

RIGHT PANEL
  - File tree
  - Shows E:\ folder structure
  - Read-only navigation


CHAT WITH AI
============

1. Click in the left panel text box
2. Type your question or command
3. Press Ctrl+Enter OR Shift+Enter
4. Wait for response (2-10 seconds depending on question)
5. Repeat


TROUBLESHOOTING
===============

"Ollama offline" error
  → Make sure Ollama is running
  → Command: ollama serve
  → Check it's on localhost:11434
  → Try: curl http://localhost:11434/api/tags

"Proxy not responding" error
  → Node.js might not be installed
  → Install from nodejs.org
  → Add to PATH
  → Restart dashboard

Chat box does nothing
  → Check that Ollama is running
  → Check browser console (F12) for errors
  → Make sure you pressed Ctrl+Enter, not just Enter

Browser shows blank page
  → Try refreshing (F5)
  → Clear cache (Ctrl+Shift+Delete)
  → Try different browser


FILES EXPLAINED
===============

dashboard.bat
  - Quick launcher from E:\ root
  - Double-click to start everything
  - Runs launch.bat from dashboard-app folder

launch.bat (in dashboard-app)
  - Starts Node proxy
  - Opens dashboard in browser
  - This is what you actually run

index.html
  - The dashboard UI itself
  - Dark GitHub-style interface
  - Chat, files, status display

proxy.js
  - Node.js CORS wrapper
  - Connects to Ollama on localhost:11434
  - Runs on localhost:9001
  - Translates browser requests to Ollama API

CONFIG.env
  - Configuration file (optional to edit)
  - Points to Ollama, proxy port, etc.
  - All paths are E: drive only

package.json
  - Node.js metadata
  - Minimal - no npm install needed

WHAT HAPPENS STEP BY STEP
=========================

When you click launch.bat:

1. Terminal opens showing setup
2. Node proxy starts (localhost:9001)
3. Browser launches with dashboard
4. You see three panels (chat, status, files)
5. Type in left panel chat box
6. Hit Ctrl+Enter
7. Proxy forwards to Ollama on localhost:11434
8. Qwen model processes your question
9. Response comes back through proxy
10. Appears in chat panel
11. You can continue chatting

ADVANCED: RUNNING WITHOUT BATCH FILE
=====================================

If .bat files don't work on your system:

1. Open PowerShell or Command Prompt
2. Type: cd E:\dashboard-app
3. Type: node proxy.js
4. Leave that terminal running
5. Open browser: file:///E:/dashboard-app/index.html
6. Done - now you can chat


ADVANCED: CHANGING OLLAMA MODEL
================================

Current model: qwen2.5:7b

To use different model:

1. Pull it: ollama pull <model-name>
2. Edit index.html
3. Find: const MODEL = 'qwen2.5:7b'
4. Change to: const MODEL = 'your-model-name'
5. Save and refresh dashboard

Available models (if pulled):
  - qwen2.5:7b
  - qwen3.5-9b
  - qwen2.5-coder:7b
  - mistral:latest
  - neural-chat:latest


GETTING HELP
============

If something breaks:

1. Check error messages in browser (F12 console)
2. Make sure Ollama is running: ollama serve
3. Try restarting the dashboard: close and run launch.bat again
4. Try a different browser
5. Check that Node.js is installed: node --version
