## Security Guarantees

### What's Protected ✓
- Models cannot be stored in app folder
- All paths validated before use
- HTTP 403 returned for violations
- Environment isolation enforced

### What's External ✓
- Models stored in `E:\models`
- Cache stored in `E:\cache`
- Ollama metadata in `E:\ollama-data` (optional)

### What's Allowed ✓
- Scan `E:\models` and subdirectories
- Import GGUF files from `E:\models`
- Use external models in queries
- Delete/move models anytime

## Daily Workflow

### Each Day
1. Set: `OLLAMA_MODELS=E:\models` (if not permanent)
2. Start: `ollama serve`
3. Start: `start-dashboard-isolated.bat`
4. Open: `http://localhost:8080`
5. Use dashboard normally

### Adding Models
1. Copy `.gguf` files to `E:\models\`
2. Dashboard → "📦 Models" → "🔍 Scan"
3. Import discovered models
4. Use in chat

### Removing Models
1. Delete `.gguf` from `E:\models\`
2. Restart Ollama (optional)
3. Model no longer available

## Troubleshooting Checklist

| Issue | Check | Solution |
|-------|-------|----------|
| Models not found | `E:\models` exists? | Create directory |
| Path validation error | `OLLAMA_MODELS` set? | Set environment variable |
| Ollama unavailable | `ollama serve` running? | Start Ollama |
| Port conflict | Other apps using 8080/9001? | Change ports or stop conflicting app |
| Permission denied | Write permission to `E:\models`? | Grant folder permissions |
| Models disappeared | Ollama restarted? | Models stay in `E:\models` |
