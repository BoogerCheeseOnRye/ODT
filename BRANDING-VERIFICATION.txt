## What Users See

**Before:**
- Dashboard title: "TEoAAAG - Minimal Dev Environment"
- Setup stored in: `%USERPROFILE%\.teoaaag\`
- Models in: `E:\teoaaag-data\models\`

**After:**
- Dashboard title: "ODT - Omni Development Terminal"
- Setup stored in: `%USERPROFILE%\.odt\`
- Models in: `X:\odt-data\models\` (X = user-selected drive)

## Migration Path

Users upgrading from old version:

1. Old config at `.teoaaag\` is NOT imported
2. First launch shows setup wizard
3. User selects drive
4. New config created at `.odt\`
5. Fresh start - works perfectly

This is intentional to keep configurations clean and separate.
