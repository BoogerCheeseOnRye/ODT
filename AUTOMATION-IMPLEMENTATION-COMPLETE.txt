## Key Features

### 1. Execution Strategies (Pick One)

| Strategy | Use Case | Speed | Consensus |
|----------|----------|-------|-----------|
| **Parallel** | Fast execution | ⚡⚡⚡ | Yes |
| **Sequential** | Build on output | ⚡⚡ | Output chain |
| **Hierarchical** | Complex plans | ⚡ | Voting required |

### 2. Safety Guardrails

✅ **Dangerous Pattern Detection**
- `rm -rf` blocks
- Database operations
- Privilege escalation

✅ **Whitelist System**
- Restrict file modifications
- Pattern-based wildcards
- Deny-by-default model

✅ **Approval Gates**
- User confirmation dialogs
- Rich violation details
- Reject/approve controls

✅ **Token Limits**
- Per-operation caps
- Rate limiting
- Duration cutoffs

### 3. Real-Time Monitoring

**Live Dashboard Shows:**
- ⏱️ Elapsed time
- 📈 Total tokens + throughput
- ✅✅ Tasks completed/failed
- 🤖 Active models
- 🔒 Safety violations
- ⚠️ Pending approvals

**Configurable Metrics:**
- Toggle each metric on/off
- Per-session settings
- Persistent selections

### 4. Intelligent Coordination

**Consensus Model**
- 2/3+ of models must agree
- Prevents single model errors
- High confidence output

**Voting Model**
- Majority wins
- Faster than consensus
- Good for non-critical tasks

**Sequential Model**
- One model, then next
- Output becomes input
- Builds on prior work

### 5. Comprehensive Logging

✅ **Real-Time Streaming**
- Logs appear as they happen
- Color-coded by type
- Auto-scrolling viewer

✅ **Filterable Logs**
- All, Info, Warnings, Errors
- Separate buttons
- Dynamic filtering

✅ **Searchable Exports**
- Full JSON export
- Timestamps, sources, messages
- Complete execution trace

### 6. Production Reports

**Summary Includes:**
- Duration + tokens used
- Success rate %
- Per-model performance
- Safety violations
- Task breakdown

**Export Options:**
- Download as JSON
- Full execution logs
- Model stats
- Timestamps

  1. Design API specification
  2. Implement REST endpoints
  3. Add authentication
  4. Write comprehensive tests
  5. Create API documentation

  T+0s:   [PLANNER] Planning workflow
  T+5s:   [CONSENSUS] 2/3 models agree on spec ✓
  T+15s:  [API] 3 models implement endpoints in parallel
  T+30s:  [AUTH] Sequentially add auth layer
