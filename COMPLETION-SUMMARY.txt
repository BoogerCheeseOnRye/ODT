## What Changed

### Security Model

**Before:**
- ❌ Models could be stored in app folder
- ❌ No path validation
- ❌ Risk of app bloat

**After:**
- ✓ Models ONLY in `E:\models`
- ✓ Path validation on all operations
- ✓ App folder stays clean
- ✓ HTTP 403 if wrong path
- ✓ Auto-creates directories

### Architecture

```
Requests to dashboard
    ↓
proxy.js / server.js
    ↓
isValidModelPath(path) checks:
    - ✓ Path in E:\models? → ALLOW
    - ✗ Path in E:\dashboard-appv2? → REJECT (HTTP 403)
    ↓
Ollama (via OLLAMA_MODELS env var)
    ↓
Models stay in E:\models
```

## How It Works

### Path Validation

```javascript
function isValidModelPath(filePath) {
    const normalized = path.normalize(filePath);
    
    // REJECT if path is inside app directory
    if (normalized.startsWith(APP_DIR)) return false;
    
    // ACCEPT if path is in models directory
    if (normalized.startsWith(MODELS_DIR)) return true;
    
    return false;
}
```

**Applied to:**
- `/api/scan-models` - Only scans `E:\models`
- `/api/import-model` - Validates before importing
- Directory creation - Creates `E:\models` if missing

### Environment Variables

## Quick Start

### 1. First-Time Setup (One time)

```powershell
# A. Set Ollama environment (permanent)
# Settings → Environment Variables
# ADD: OLLAMA_MODELS = E:\models
# ADD: OLLAMA_CACHE = E:\cache
# Restart Ollama

# B. Create directory
mkdir E:\models
```

### 2. Daily Usage

```batch
# Terminal 1: Ollama
ollama serve

# Terminal 2: Dashboard
cd E:\dashboard-appv2
start-dashboard-isolated.bat
```

Then visit: **http://localhost:8080**

### 3. Add Models

1. Copy `.gguf` files to `E:\models\`
2. Dashboard → "📦 Models" → "🔍 Scan E:\models"
3. Click "Import" on discovered model
4. Done! Model is now available
